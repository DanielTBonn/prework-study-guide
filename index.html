HTML

  • The <!DOCTYPE html> element tells the browser that this document is written in html.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The meta element defines metadata such as character set, description, keywords, author, and viewport.
  • The link element connects an external resource to the HTML document.
  • The base element defines the base URL for a webpage.
  • The style element defines the CSS styles associated with a website.
  • An element refers to the entire element from opening tag to closing tag. and a tag refers only to what appears inside the angle brackets. They are used interchangeably.
  • Elements nested within other elements are called child elements. The element the children are nested in is said to be the parent element.
  • The a element, which stands for anchor element, creates links to the same webpage or other webpages.
  • The section element allows us to organize the content on our page based on our topics.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A period "." preceding a selector is designated as a class selector.
  • A border property assigns a border to your selector.
  • A box-shadow property creates an offset around your elements frame (x-offset, y-offset, blur, spread-radius, color).
  • A link element nested within the head element is used to link your style.css file to your html file.
  • Assigning colors in CSS can be done in many ways including semantic reference (gray) and hexadecimal (#A1B2C3).

Git

  • git status: checks what branch we are currently on.
  • git checkout -b 'branch-name': creates a new branch and switches to it.
  • git pull origin main: pulls in up to date code from the current main branch.
  • git push origin 'branch-name': pushes up all changes to the main branch.
  • git add -A: stages changes of all files in current working branch.
  • git commit -m "commit-changes": commits changes to our repository adding the change descritpion within the quotes.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Conditional statements and loopsinterrupt control flow by controling the order in which code is executed.
  • Functions are sets of instructions that tells the computer how to perform a certain task.
  • The () following a function name are a place we define parameters.
  • The method console.log() outputs a message to the web console by adding an argument between the parentheses.
  • We connect .js files to our .html files by using the script element and setting its src attribute to the JS file path.
  • We can see JS changes in the browser by using Chrome DevTools.
